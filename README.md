# city-geo-history-research v1.3

面向城市旅行、地区研究、地理历史科普、旅行拍摄与城市文化 Vlog 的 Codex / Claude Code skill。

它不是普通景点清单工具，而是一套完整的城市理解与旅行执行系统：

```text
查资料 → 拆地理因子 → 建知识包 → 生成旅行方案 → 校验时间预算体力 → 安排拍摄 → 拆镜头 → 写脚本 → 核查事实与动态信息
```

## v1.3 新增重点

v1.3 在 v1.2 深层地理因子版上增加了“旅行执行层”：

- 旅行约束输入
- 最合适的住宿区域判断
- 逐日时间轴
- 景点间交通与停留时间
- 预算范围估算
- 步行、爬坡和体力校验
- 天气、开放时间、预约和门票实时核查
- 雨天、闭馆、人多和时间不足替代方案
- 旅行拍摄任务、短视频选题和长 Vlog 主线

## v1.3 三个旅行模式

### 1. 旅行攻略

```text
/旅行攻略 南京，两天一夜，双人，预算2500元，不含往返大交通，节奏松弛，兴趣是历史和城市空间
```

别名：

```text
/一日游攻略
/两天一夜攻略
/城市路线规划
```

输出包括：

- 方案假设
- 最佳季节和气候体感
- 住宿区域
- 每日时间轴
- 交通和停留时间
- 地理历史知识点
- 餐饮区域
- 预算和体力
- 雨天与闭馆替代
- 动态信息核查清单
- 路线可行性评分

### 2. 旅行拍摄攻略

```text
/旅行拍摄攻略 青岛，一天，设备是 iPhone 15 Pro Max + DJI Pocket 3，重点拍海岸、山地、老城坡度、海风和情侣生活感
```

它包含普通旅行攻略全部内容，并增加：

- 每日内容主题
- 必拍镜头
- 景别和运镜
- 光线和环境声
- 短视频选题
- 长 Vlog 章节
- 人物互动位置
- 封面候选画面
- 素材冗余方案

### 3. 旅行实时核查

```text
/旅行实时核查 济南，明天出发，计划去趵突泉、大明湖、山东博物馆和老商埠
```

用于出发前 1—3 天核查：

- 天气与灾害预警
- 开放时间
- 预约政策
- 门票
- 地铁和公交调整
- 封路、施工和大型活动
- 临时闭馆和限流

此模式必须联网查询；无法联网时不会假装完成实时核查。

## 路线编排原则

- 同一区域尽量安排在同一天
- 不频繁横穿城市
- 必须计算交通、排队、吃饭、休息和找路时间
- 每天保留 60—90 分钟机动时间
- 普通节奏默认不安排明显特种兵路线
- 每天给出可删地点
- 体力过高时主动删减，而不是继续塞景点
- 旅行体验优先于收集无用素材

## 静态知识与动态信息

### 静态知识

- 地理位置
- 地形水系
- 历史沿革
- 城市空间结构
- 建筑、饮食和产业形成机制

### 动态信息

- 天气
- 门票
- 开放和闭馆
- 预约规则
- 交通班次
- 酒店与餐厅价格
- 临时施工和限流

动态信息必须实时查询并注明查询日期。无法确认时必须标注未核实。

## 资料可信度

### A 级

政府、文旅、博物馆、文保单位、地方志、学术机构、气象、水利、自然资源、生态环境、官方规划和地图。

### B 级

主流媒体、百科、官方公众号、有清楚出处的地方历史文章和数据整理。

### C 级

自媒体、论坛、小红书、抖音笔记、未注明来源的故事、未注明年份的价格和排名。

C 级只能作为体验线索，必须标注待核实。

## 验收标准

一份合格攻略必须：

- 时间上走得完
- 地理上少折返
- 预算有范围
- 体力有校验
- 动态信息有核查状态
- 有雨天、闭馆或人多替代
- 每个地点说明为什么值得去
- 拍摄型攻略对应具体镜头和内容

只列景点、餐厅和一句话介绍，不合格。
