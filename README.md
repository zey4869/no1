# city-geo-history-research v1.1

面向城市旅行、地区研究、地理历史科普、城市文化 Vlog 的 Codex / Claude Code skill。

它的目标不是生成普通旅游攻略，而是把一个城市或地区整理成可用于内容创作的「城市内容生产资料包」。

v1.1 已从「城市知识包」升级为完整前期系统：

```text
查资料 → 建知识包 → 出选题 → 排拍摄 → 拆镜头 → 写脚本 → 核资料
```

## 适合谁

- 城市旅行创作者
- 地理历史科普创作者
- 本地生活内容创作者
- 城市文化 Vlog 创作者
- 准备去某地拍摄前需要系统资料的人
- 想把地方志、博物馆、文保资料转成短视频的人

## 它能输出什么

- 城市知识包
- 地区历史地理资料包
- 景区/街区知识包
- 城市冷知识选题库
- 竖屏短视频选题
- 横屏中视频主线
- 长 Vlog 主线
- 拍摄执行路线
- 逐镜头脚本
- 90 秒旁白
- 剪映字幕分行版
- 封面标题
- 发布文案
- 资料可信度核查

## 资料可信度规则

### A 级资料：优先使用

- 政府官网
- 文旅局官网
- 博物馆、纪念馆、文保单位官网
- 地方志、县志、市志
- 国家文物局、世界遗产中心、联合国教科文组织资料
- 学术论文、大学/研究机构资料
- 正规出版社书籍
- 官方地图、规划文件、历史地图

### B 级资料：辅助使用

- 百科类资料
- 主流媒体报道
- 城市宣传片、官方公众号
- 地方历史研究文章
- 高质量旅行记录

### C 级资料：谨慎使用

- 自媒体文章
- 论坛帖子
- 小红书/抖音笔记
- 未注明来源的故事
- 民间传说、坊间说法

C 级资料必须标注「待核实」或「民间说法」。

## 核心原则

1. 不做普通攻略，要解释城市为什么形成今天的样子。
2. 不只罗列景点，要建立地理、历史、空间、文化之间的关系。
3. 不把百科当唯一来源，要优先使用政府、博物馆、地方志、学术、文保资料。
4. 每个知识点都要考虑是否能转化成镜头、选题、旁白或拍摄路线。
5. 结论要分可信度：高、中、低、待核实。
6. 不确定的历史说法不要硬写成结论。
7. 输出必须服务于拍摄、脚本、选题或资料核查。
