# city-geo-history-research v1.2

面向城市旅行、地区研究、地理历史科普、城市文化 Vlog 的 Codex / Claude Code skill。

它的目标不是生成普通旅游攻略，而是把一个城市或地区整理成可用于内容创作的「城市内容生产资料包」。

v1.2 从「城市知识包 + 内容生产流程」继续升级为 **深层地理因子版**：

```text
查资料 → 拆地理因子 → 建知识包 → 出选题 → 排拍摄 → 拆镜头 → 写脚本 → 核资料
```

## v1.2 新增重点

过去地理部分容易停留在：

```text
位置、地形、水系
```

v1.2 扩展为 12 个深层地理因子：

```text
区位
地形地貌
气候
水文
土壤与农业
生态与植被
资源与产业
灾害与风险
交通通道
城市微气候
生活方式与文化适配
影像可视化
```

这能解释一个城市更深层的问题：

- 为什么城市建在这里？
- 为什么向这个方向扩张？
- 为什么这里形成某种饮食、建筑、产业和性格？
- 为什么这座城市有湿热、干冷、海风、树荫、泉水、坡地、商贸或物流气质？
- 哪些地理因素能转化成镜头、选题和旁白？

## 资料可信度规则

### A 级资料：优先使用

- 政府官网
- 文旅局官网
- 博物馆、纪念馆、文保单位官网
- 地方志、县志、市志
- 国家文物局、世界遗产中心、联合国教科文组织资料
- 学术论文、大学/研究机构资料
- 正规出版社书籍
- 官方地图、规划文件、历史地图
- 气象局、水利部门、自然资源部门、生态环境部门、地震局等专业机构资料

### B 级资料：辅助使用

- 百科类资料
- 主流媒体报道
- 城市宣传片、官方公众号
- 地方历史研究文章
- 高质量旅行记录
- 非官方但有清楚出处的数据整理

### C 级资料：谨慎使用

- 自媒体文章
- 论坛帖子
- 小红书/抖音笔记
- 未注明来源的故事
- 民间传说、坊间说法
- 未注明数据年份的气候、人口、排名、称号资料

C 级资料必须标注「待核实」或「民间说法」。

## 核心原则

1. 不做普通攻略，要解释城市为什么形成今天的样子。
2. 不只罗列景点，要建立地理、气候、历史、空间、文化之间的关系。
3. 不把百科当唯一来源，要优先使用政府、博物馆、地方志、学术、文保、气象、水利、自然资源资料。
4. 每个知识点都要考虑是否能转化成镜头、选题、旁白或拍摄路线。
5. 结论要分可信度：高、中、低、待核实。
6. 不确定的历史、地理、气候数据不要硬写成结论。
7. 输出必须服务于解释、拍摄、脚本、选题或资料核查。
8. 对“第一、唯一、最大、最早、世界级”等说法必须谨慎核查。
