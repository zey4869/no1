# city-geo-history-research v1.1 快捷调用手册

## 1. 一句话记忆

```text
查资料 → 建知识包 → 出选题 → 排拍摄 → 拆镜头 → 写脚本 → 核资料
```

这个 skill 不是普通旅游攻略工具，而是城市内容创作前期系统。

---

## 2. 基础调用

```text
/城市研究 南京
```

快速了解一个城市的地理、历史、文化、空间结构和内容选题。

```text
/城市知识包 南京
```

输出完整城市知识包，适合旅行前系统准备。

```text
/地区历史地理 临沂兰山区
```

研究区县、城区、街区、乡镇等更小尺度地区。

```text
/城市冷知识 青岛
```

获取可做短视频的地理历史冷知识。

```text
/城市故事文案 南京
```

把资料转成有叙事感的短视频或中视频文案。

---

## 3. v1.1 六个子模式

### 3.1 资料采集

```text
/城市资料采集 南京
```

适合正式写脚本前，先查清楚资料。输出资料目录、资料优先级、关键词、可转化事实和待核实问题。

---

### 3.2 选题生成

```text
/城市选题生成 南京
```

适合把资料转成短视频、中视频、长 Vlog 选题库。

推荐加强版：

```text
/城市选题生成 南京，重点关注六朝、明城墙、民国、秦淮河、法国梧桐、城市破碎感。请给我 10 条竖屏短视频、3 条横屏中视频、1 条长 Vlog 主线，并按可拍性排序。
```

---

### 3.3 拍摄执行

```text
/城市拍摄执行 南京，两天一夜，设备是 iPhone 15 Pro Max + DJI Pocket 3
```

适合出发前安排拍摄路线、必拍镜头、备用镜头、素材冗余策略。

推荐加强版：

```text
/城市拍摄执行 南京，两天一夜，设备是 iPhone 15 Pro Max + DJI Pocket 3，重点拍城市地理、历史、情侣隐线和生活感。请给我路线、必拍镜头、备用镜头和每个镜头对应的选题。
```

---

### 3.4 镜头拆解

```text
/城市镜头拆解 明孝陵，90秒，主题是南京的厚重感
```

适合把一个地点、主题或选题拆成逐镜头表。

输出包括：时间、画面、景别、运镜、旁白/字幕、知识点、备注。

---

### 3.5 脚本生成

```text
/城市脚本生成 南京城墙，90秒，风格真实细腻，有知识有情绪
```

适合生成完整旁白、字幕分行版、封面标题、发布文案、评论区引导。

推荐加强版：

```text
/城市脚本生成 南京城墙为什么让南京显得厚重，90秒。请按前2秒钩子、3-20秒建立问题、21-45秒解释原因、46-70秒现实体验、71-90秒收束互动来写，并给我剪映字幕分行版。
```

---

### 3.6 资料核查

```text
/城市资料核查 南京城墙是世界第一大城墙这个说法
```

适合检查某个历史说法、脚本表达或标题是否稳妥。

推荐加强版：

```text
/城市资料核查 我准备在视频里说“南京城墙是世界第一大城墙”，请拆解事实、判断可信度、指出风险，并给我更安全的旁白表达。
```

## 4. 适合你的使用方式

### 旅行前 7 天

```text
/城市资料采集 目的地城市
/城市知识包 目的地城市
```

目标：先建立整体理解和资料来源。

### 旅行前 3 天

```text
/城市选题生成 目的地城市
/城市拍摄执行 目的地城市，两天一夜
```

目标：决定拍什么，以及怎么拍。

### 拍摄前 1 天

```text
/城市镜头拆解 具体地点，90秒，主题是 xxx
```

目标：现场不迷路，知道每个镜头服务哪条内容。

### 剪辑前

```text
/城市脚本生成 主题，90秒，结合我已经拍到的素材：xxx
```

目标：把素材变成脚本。

### 发布前

```text
/城市资料核查 你准备写进视频里的关键说法
```

目标：避免历史、地理、称号、年份说错。

## 7. 推荐完整工作流

```text
第一步：/城市资料采集 城市名
第二步：/城市知识包 城市名
第三步：/城市选题生成 城市名
第四步：/城市拍摄执行 城市名 + 时间 + 设备
第五步：/城市镜头拆解 具体主题
第六步：/城市脚本生成 具体主题 + 时长 + 风格
第七步：/城市资料核查 关键历史/地理说法
第八步：用 cheat-on-content 给脚本打分、预测和复盘
```
